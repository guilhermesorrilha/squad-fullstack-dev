# Step 11: Aprovar QA

Checkpoint de aprovação do relatório de QA. Se o QA reportou FAIL, o usuário decide se corrige ou avança.

## Perguntas

### Pergunta 1: Resultado do QA
Revise o relatório de QA acima. Como deseja prosseguir?

1. Aprovar — bugs são aceitáveis, seguir para auditorias
2. Corrigir — voltar para Frontend/Backend para corrigir bugs reportados
3. Pular bugs minor — corrigir só critical/major e avançar
