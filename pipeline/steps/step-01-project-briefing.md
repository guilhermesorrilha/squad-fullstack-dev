# Step 01: Project Briefing

Checkpoint inicial onde o usuário descreve o projeto que será desenvolvido pelo squad.

## Perguntas

### Pergunta 1: Descrição do Projeto
Descreva o projeto que você quer desenvolver. Inclua:
- O que o sistema/site deve fazer (funcionalidades principais)
- Quem vai usar (público-alvo)
- Alguma restrição técnica ou preferência de stack

Exemplos:
- "Um dashboard SaaS para gerenciar assinaturas de clientes, com login OAuth, painel de métricas e integração com Stripe"
- "Uma landing page para meu produto digital com formulário de captura, seção de depoimentos e checkout integrado"
- "Um portal interno para a equipe de vendas acompanhar leads, com filtros avançados e exportação para Excel"

### Pergunta 2: Prioridade
Qual é a prioridade principal deste projeto?
1. Velocidade de entrega (MVP rápido)
2. Qualidade e robustez (produção-grade)
3. Escalabilidade (preparar para crescimento)
4. Segurança (dados sensíveis ou compliance)
